# The Body — Information Domain

The body is the whole planetary network.
All bioregions.
All organs.
All tissues.

Information at this scale needs standards.
Without standards nothing connects.

A standard is an agreement.
It says this format means this thing.

The knowledge commons holds standards.
It holds best practices.
It holds failure reports.

Every organ contributes what it learns.
Every organ takes what others have learned.

The knowledge graph grows.
Each node is a concept.
Each edge is a relation.

A concept has a name.
A concept has a definition.
A concept has evidence.
A concept has a confidence level.
A concept has a source.

An edge has a from concept.
An edge has a to concept.
An edge has a type of relation.
An edge has a strength of evidence.

The graph is queried.
A question comes in.
The graph returns relevant concepts.

A decision support query example.
What is the best water system for clay soil in a dry climate.

The graph answers with concepts that match.
It answers with relations that connect them.
It answers with confidence levels.

Standards at the body level include data formats.
Standards at the body level include measurement units.
Standards at the body level include safety thresholds.

Data format standard uses JSON for sensor readings.
It uses Parquet for large datasets.

Measurement unit standard uses SI units.
Liters. Kilograms. Meters. Seconds. Celsius.

Safety threshold standard uses maximum contaminant levels.
These do not vary by region.

The knowledge commons is open.
Read by anyone.
Write by contributors.

Contributors are verified.
Each contribution is reviewed.
Accuracy matters.

False information is flagged.
Repeated false information gets a contributor banned.

The commons has a decay rule.
Old information gets flagged.
It must be reviewed.

Standards evolve.
New information updates the graph.
Old numbers get replaced.

The commons runs on open source.
The code is open.
The data is open.

No corporation owns the commons.
No government owns the commons.
The users own it.

The commons is funded by contribution.
Each organ contributes time or goods annually.

Governance of the commons falls to a council of bioregion representatives.
They set standards.

Standards changes go through a proposal process.
Comment period. Revision. Final vote.

The body knows what each organ knows.
It sees patterns across bioregions.
It sees which water systems work in dry climates.
It sees which fail.
It sees which soil methods work in clay.
It sees which fail.

It sees correlations no single organ can see.
It connects distant experiments.

A desert organ learns from a tropical organ.
A cold organ learns from a warm organ.
The graph makes this possible.

The body runs on the same architecture as the organ.
It scales up.
The inference engine at body level is distributed.
It runs across many servers.

Each server holds a partition of the graph.
They sync with each other.

The body does not have one central brain.
It has many nodes.
They cooperate.

Query routing finds the relevant nodes.
It returns results from across the network.

The body has memory.
It remembers what worked.
It remembers what failed.
It forgets nothing.
Old failures stay visible.
They warn new experiments.

This is collective knowledge.
It belongs to all.
It serves all.

Bioregions contribute their experiments.
They take from the commons what they need.

A bioregion starting a water project queries the graph.
It finds similar projects.
It finds what worked.
It finds what to avoid.
It saves time and resources.

The graph supports decisions.
It does not make decisions.
People make decisions.

The graph provides evidence.
It provides context.
People provide judgment.
