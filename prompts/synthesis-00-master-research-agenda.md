# Synthesis Prompt: Master Research and Systems Creation Agenda

## Purpose

This document coordinates all five research prompts into a unified research agenda. It is the meta-layer that ensures the five streams of research produce a coherent synthesis, not five isolated reports.

## The Five Research Streams

```
Stream A: Emergence Methodology
    └── Focus: How does change actually happen in complex community systems?
    └── Key tension: Engineered change vs. Emergent change
    └── Output: Emergence invitation protocol, field readiness diagnostic

Stream B: Relational Values Framework
    └── Focus: What values vocabulary is adequate for regenerative community design?
    └── Key tension: Western individualist framing vs. Relational/Indigenous framing
    └── Output: Value word audit, cross-framework translation matrix, sacred protocol

Stream C: Values-Architecture Bridge
    └── Focus: How do you trace decisions back to the values that motivated them?
    └── Key tension: Values determine structure vs. Structure determines values
    └── Output: ADR with values tracking, conflict as values collision protocol

Stream D: Ecological Stakeholder Integration
    └── Focus: How does the ecosystem have voice in community governance?
    └── Key tension: Human-centric values vs. Intrinsic value of non-human world
    └── Output: Non-human stakeholder matrix, ecological indicator dashboard, intergenerational representation

Stream E: Process Methodology
    └── Focus: How do you select and sequence interventions appropriately?
    └── Key tension: Sequential process vs. Recursive, state-contingent response
    └── Output: Intervention taxonomy, facilitator vs. architect framework
```

## The Integration Problem

Each research stream produces findings in isolation. But the five streams are not independent — they interact:

```
Stream A (Emergence) → determines WHEN to apply which intervention (E)
Stream B (Values) → determines WHAT vocabulary guides the process (A, C, D)
Stream C (Bridge) → determines HOW values are embedded in decisions (D, E)
Stream D (Ecological) → determines WHAT the community is IN SERVICE OF (B, C)
Stream E (Process) → determines WHO does what WHEN (A, B, C, D)
```

## Cross-Stream Integration Questions

### Q1: Which stream leads in a given community context?
Not all communities need all five streams activated equally.
- A new community with high trust and unclear direction → lead with Stream A (emergence invitation)
- A community with conflict → lead with Stream C (values collision surfacing)
- A community on degraded land → lead with Stream D (ecological restoration as priority)
- A community with extractive values vocabulary → lead with Stream B (relational values re-framing)

**Deliverable: CONTEXT-STREAM MAPPING — A decision guide for which research stream to activate given community state**

### Q2: How do the five streams share a common ontology?
Each stream will develop its own vocabulary. But the methodology needs shared terms.
- What is the shared term for "the thing we are trying to change"? (Field? System? Community? Land?)
- What is the shared term for "the capacity to act"? (Agency? Power? Authority? Voice?)
- What is the shared term for "what matters"? (Values? Needs? Meanings? Obligations?)

**Deliverable: SHARED ONTOLOGY GLOSSARY — Common terms across all five streams, with explicit notes on where each stream's usage differs from the shared term**

### Q3: What is the minimum viable version of all five streams combined?
The full research agenda is large. But a community cannot wait for all five streams to complete.
- What is the smallest subset of findings from each stream that would enable a first iteration?
- What can be deferred to later research passes?
- What is the alpha version of the methodology — the smallest thing worth testing?

**Deliverable: MINIMUM VIABLE INTEGRATION — The core subset of all five streams that constitutes a testable methodology**

### Q4: How do you know if the synthesis is succeeding?
What are the success criteria for the integrated methodology, not just the individual streams?
- Does the methodology produce decisions that the community owns?
- Does ecological health improve (or at least not degrade)?
- Are conflicts resolved at the values level, not just at the technical level?
- Does the community become less dependent on external facilitation over time?

**Deliverable: SUCCESS CRITERIA FRAMEWORK — How to evaluate the integrated methodology's effectiveness**

## Research Execution Order

**Phase 1 (Parallel, 4 weeks):**
- Streams A, B, C, D, E all begin simultaneously
- Each stream produces its intervention type taxonomy / minimum viable output
- Streams report progress to a shared synthesis log

**Phase 2 (Integration, 2 weeks):**
- Q1-Q4 above are addressed
- Common ontology is negotiated across streams
- Minimum viable integration is defined

**Phase 3 (Alpha Build, 4 weeks):**
- Minimum viable methodology is assembled
- First community pilot is identified and briefed
- Documentation (prompts, scripts, templates) is prepared

**Phase 4 (Pilot & Iterate, ongoing):**
- Alpha methodology tested in one community context
- Findings fed back into all five streams
- Methodology revised, streams updated, repeat

## Execution Notes for Autonomous Agents

- Each research prompt file is designed to be worked independently by a capable agent
- Agents should cross-reference findings with other streams before finalizing
- When two streams produce contradictory findings, do NOT resolve by choosing one — document the contradiction as a productive tension
- The synthesis is not the sum of the parts — it is the pattern that emerges from their interaction
- Document EVERYTHING — a community that cannot see how a methodology was built cannot own it
